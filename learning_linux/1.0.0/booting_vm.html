Booting the Kernel directly in a Qemu Virtual Machine

To boot the x86_64 Linux kernel in a virtual machine, we need a rootfs prepared for the x86_64 architecture. We may use a prebuilt rootfs available for download over the internet or we may build a custom rootfs of our own.

To generate a custom rootfs, follow the document Building Custom Rootfs using The Yocto Project. This will generate a qcow2 image that represents a hard disk installed with rootfs as well as the linux kernel. You can peek into the contents of the hard disk image you may mount the image as a loopback device as described in Mount, inspect, modify a qcow2 image.
